起因

使用 hexo 2.x 有两年多了，也改了一个主题自用着，但一直未去填坑完善主题，里面的代码也较多冗余，拖拖拉拉，兴（喜）趣（新）不（厌）高（旧）。

随着 hexo 3.x 版本的发布，其使用文档已相当健全了，翻了一下，其模块更独立化、 api 更简洁，可以很方便地建立新的主题，便动了再造轮子的想法。

上周周末干脆重新安装了 hexo：由 nvm 安装 npm， 再由 npm 安装 hexo 及 hexo 的各个模块（各个模块独立后，可按需安装）。这次指定了安装目录，统一了目录，后续删除就方便了。

这周大（粗）刀（鄙）阔（简）斧（陋）地完成了我的第二个主题，开发代号为 flexin，名称为 blueberry。 至此，闲置半年的记录博客习惯预计将会恢复。

主题简介

理念

  1. 更关注内容和文章质量，去掉周边功能，比如统计、搜索
  2. 类 unix 文本的直白展示方式，代码不加高亮
  3. 关注阅读体验，关注字体，优先使用开源的 open sans 无衬线字体
  4. 打开要快，使用国内 cdn 的源来加载 jquery，同时不再使用前端框架 Bootstrap，不使用图标字体

功能

  1. 首页做成归档的形式，大字报列出最近几篇文章
  2. 首页的每个文章标题下方，又列出文章生成时间、所属目录、所打标签，这也是我管理内容的三个维度
  3. 把菜单链接放到页面右下角，减少阅读干扰
  4. 分页功能，下一页，或下一篇文章
  5. 文章右上角增加目录
  6. 图片居中

特色

  1. 不支持 IE
